概览

Dashboard 是一个 WebRTC 会话实时监控与 CSV 数据分析工具，通过代理连接到本地 WebRTC 服务（默认 https://localhost:8080）。页面每 1 秒自动刷新，布局分为：实时监控区（上半部分）和 CSV 离线分析区（下半部分）。

上半部分 — 实时监控

Service Origin / Room ID
指定 WebRTC 服务的地址和要监控的房间号。修改 Room ID 后自动切换房间。
检查服务
检测 WebRTC 服务是否在线，获取当前所有房间概况。
清理当前 Room Stats
清空当前房间的所有统计数据（含历史），一般用于一轮测试结束后重置数据。

Summary Strip（5 个指标卡）：Rooms（当前房间数）、Stats（统计状态）、Mesh（拓扑状态）、CSV（分析状态）、NACK（NACK 模式/次数）。

Peer Pairs：当前房间内所有观察到的 Peer 对（Peer → Remote）。Mesh Topology：每个 Peer 对的连接状态、RTT、Loss、Bitrate。Latest Stats：最新一条样本的全部指标详情。Stats History：最近 20 条历史样本的滚动表格。

下半部分 — CSV 离线分析

支持两种 CSV 加载方式：

方式 1 — 上传本地 CSV 文件
点击文件选择框选取一个或多个 .csv 文件，选择对比指标（RTT / Loss / Jitter / Bitrate / FPS），点击 Analyze CSV 进行分析。
方式 2 — 加载服务端 Session CSV
点击 Load Sessions 从 WebRTC 服务拉取已完成（Finished）的测试会话列表，在下拉框中选择后点击 Load Session CSV 加载并分析。

对比表格：展示每个 CSV 的基本信息（文件名、样本数、Room、Session、Peer、Remote）和各指标的平均值（Avg）、最小值（Min）、最大值（Max）。

趋势图：以折线图叠加所有 CSV 的指标随时间（样本序号）变化的趋势，下方显示最佳值和图例。

指标选择：切换 csvMetricSelect 下拉框可更换趋势图和最佳值计算指标。方向：RTT/Loss/Jitter 越小越好 → Best = min；Bitrate/FPS 越大越好 → Best = max。

数据分析思路

  1. 建立基线：用 NACK=on 的默认配置跑一轮测试，记录 RTT、Loss、Bitrate、FPS 的基线值。
  2. 单变量对比：每次只改变一个变量（如 NACK on→off、bitrate 300→800 kbps、ABR off→on），保持其他条件不变。
  3. 关注趋势而非单点：趋势图展示的是整个 session 过程中指标的变化，可以观察到启动阶段、稳定阶段、劣化阶段的特征。好的方案应该有稳定的曲线。
  4. 多维度交叉：不要只看一个指标。例如：高 Bitrate 如果伴随高 Loss 则实际质量可能更差。建议同时对比 RTT + Loss + Bitrate。
  5. 使用 Session 功能：在 RTCTraining 页面用 Start Session / Finish 记录完整会话，然后在 Dashboard 通过 Load Sessions 加载，可以系统性地对比多次实验。
